HashMap和HashTable的区别

HashMap和HashTable都实现了Map接口，主要的区别有：线程安全性，同步以及阻塞。

  1. HashMap几乎是等价于HashTable的，但是HashMap是非synchronized的，并且接受null值（即null的key和value）
  2. HashMap是非线程安全的，HashTable是线程安全的，多个线程可以共享HashTable；如果没有正确同步，多个线程是不能共享HashMap的。在Java5中提供了CurrentHashMap来替代HashTable，这比HashTable的扩展性更好。
  3. HashMap的迭代器(Iterator)是fail-fast迭代器，而Hashtable的enumerater迭代器不是fail-fast的。所以当有其他线程改变HashMap的结构时，会抛出异常。
  4. 因为HashTable是线程安全的而且也是synchronized的，因此在单线程环境下，他比HashMap慢，如果不需要同步，只需要单一的线程，那么可以选择HashMap。
  5. HashMap不能保证随着时间的推移元素的次序保持不变。
