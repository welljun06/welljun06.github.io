Spring 笔记

Posted on 2018-04-30

概念

  • Spring是一个开源框架，为了解决企业应用复杂性而创建的，现在不限于使用在企业应用上
  • 是一个轻量级的控制反转(IoC)和面向切面(AOP)的容器框架

IoC

控制权交由第三方容器，应用程序本身不负责依赖对象的创建和维护。

DI(依赖注入)是其中的一种表现形式，目的是为了创建对象并组织对象的关系，

依赖注入：在IoC容器运行的时候动态地将依赖关系注入到对象之中。

单元测试

  • 需要用到junit类
  • 创建UnitTestBase类，完成对Spring配置文件的加载和销毁
  • 所有的单元测试类都继承自UnitTestBase，通过他的getBean方法获取需要的类
  • 子类(具体执行单元测试的类)需要添加注解(@RunWith)
  • 单元测试的方法添加注解:@Test

Spring注入

是指在启动Spring容器加载bean配置的时候，完成对变量的赋值行为

  • 设值注入：相当于set方法（需要在类中实现set方法）
  • 构建注入：相当于构建函数（需要实现构造方法）

Bean

配置方式

  • 在XML中进行显性配置
  • 在Java中进行显性配置
  • 隐式的bean发现机制和自动装配

作用域

singleton：单例，指一个Bean容器中只存在一份

prototype：每次请求都会创建新的实例，destory方法不生效

request：在每次request请求中生成实例，且仅在当前request中有效

session：和request类似

globalsession：基于web中的portlet有效

生命周期

配置初始化和结束方法

  • 继承方法
  • 设置配置文件
  • 设置默认方法

Aware

容器管理的Bean一般不需要了解容器的状态和直接使用容器，但是在某些情况下，是需要在Bean中直接对IOC容器进行操作的，这时候，就需要在Bean中设定对容器的感知。Spring IOC容器也提供了该功能，它是通过特定的Aware接口来完成的。

自动装配(Autowiring)

byname：根据属性自动装配

byType：如果容器中存在一个与指定类型相同的bean，那么将与该属性自动装配；如果存在多个该类型的bean，那么则将抛出异常，并指出不能使用bytype方式进行装配，如果没有匹配到相匹配的bean，则什么事都不会发生

constructor：与byType相似，不同之处是在于使用构造器实现。如果容器中没有找到与构造器参数类型一致的bean，则抛出异常。

首先使用@Component注解表明该类为组件类，接着显示配置Spring，命令它去寻找带有@Component注解的类

Resources

  • classPath
  • file
  • http

Bean管理的注解

从Spring3.0开始可以通过Java的注解来定义bean而不用通过xml来配置

你只需要在xml里配置一个注解扫描器，把你的包名一填，然后就可以有三种方法配置bean

1、直接在private成员变量上加autowired的注解

2、写个seter方法，加上autowired注解

3、写个构造器，加上autowired注解

@Qualifier

  • 按类型自动装配可能有多个bean实例的情况，可以使用Spring的qualifier注解缩小范围，也可以用于指定单独的构造器参数或者方法参数
  • 可用于注解集合类型变量

与@Autowried结合使用注入特定的类

@Bean

  • 用于标识一个用于配置和初始化一个由SpringIoC容器管理的新对象的方法，类似与XML配置文件的
  • 可以在Spring中的@Component注解的类中使用@bean注解
  • 上一点中常常使用的是@configuration

AOP

概念

在软件开发中，散布于应用中多处的功能被称为横切关注点，这与业务逻辑是分离开的。

把这些横切关注点与业务逻辑相分离正是面向切面编程(AOP)所要解决的问题。

主要是实现横切关注点与它们所影响的对象之间的解耦。

上图表示被划分为模块的应用，每个模块的核心功能都是为特定的业务领域提供服务，这些模块都需要辅助功能，例如事务和安全管理。

术语

通知

通知也就是切面的工作，分为五种类型：

  • 前置通知(before)：在目标方法被调用前调用此通知功能
  • 后置通知(After)：在目标方法完成之后调用通知，不关心方法的输出
  • 返回通知(After-returning)：在目标方法成功执行后调用
  • 异常通知(After-returning)：在目标方法抛出异常后返回
  • 环绕通知(Around)：包裹了被通知的方法，在被通知的方法调用之前和之后执行自定义的行为。

连接点：连接点是在应用执行过程中能够插入切面的一个点。

切点：会匹配通知所要织入的一个或者多个连接点。

切面：切面是通知和切点的结合。

可以通过定义pointcut来注解频繁使用的切点表达式
