SpringBoot笔记

Posted on 2018-04-27

SpringBoot、SpringMVC、Spring的关系

Spring可以说是一个管理bean的容器，SpringMVC是其中的一个开源项目。

Spring是一个轻量级的控制反转（IoC）和面向切面（AOP）的容器框架。

Spring Boot不是一门新技术。从本质上来说，Spring Boot就是Spring，它做了一些对Spring Bean的默认配置。

SpringBoot优点

  • 化简为繁，简化配置
  • 微服务的入门级框架

启动方式

  • 直接在idea启动
  • 通过到项目文件夹使用 mvn spring-boot:run启动
  • 在项目目录直接使用mvn install打包项目后使用java -jar执行target目录下的jar包

属性配置

@Value

实现配置内容的注入

@Component

@ConfigurationProperties

@Autowried

当属性变多后，可以一次导入，只需要创建一个properties类，注解@Component，将属性通过ConfigurationProperties中的prefix导入，再通过@Autowried注入。

控制器的使用

@Contorller：处理http请求

@RestController：返回json

@RequestMapping：配置url映射，如果使用get方法，可以使用@GetMapping简化，POST也是类似

@PathVariable：获取路径中的参数

@RequestParam：获取url中的参数

spring data jpa 是spring对hibernate的封装；优点是不需要写sql，适合简单的管理系统类型的项目，缺点也是不写sql ，做复杂的互联网项目时 还是 springboot+mybatis好用。

事务管理

相当于回滚，添加在service层的方法中

两条 sql 语句同时在一个方法中执行，为了防止一个 sql 语句执行成功而另一个 sql 语句执行失败，引入了事务管理，需要在方法上加 @Transactional事务注解

事务确保了数据库数据的完整性和一致性

表单验证

  • 在需要验证的属性上添加@Min标签并配置value和message属性
  • 在Controller层的方法参数前添加@valid标签， 添加BindingResult属性
  • 判断BindingResult中的hasErrors方法，然后通过bindingResult.getFieldError().getDefaultMessage()输出之前设定的错误信息
