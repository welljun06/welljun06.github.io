Linux进程间通信

管道（PIPE）

管道可用于具有亲缘关系进程之间的通信，有名管道(FIFO)克服了管道没有名字的缺陷。因此，除了具有管道的基本功能之外，还允许没有亲缘关系的进程间的通信。这实际上是只存在于内存中的文件。

实现机制

管道是由内核管理的一个缓冲区，管道的一端连接一个进程的输出，另一端连接进程的输出。当两个进程都结束的时候，管道也会消失。

管道建立是利用fork来实现，让两个进程连接到一个管道上。先是一个进程1输入和输出都连接到一个PIPE上，然后进程1通过fork得到进程2，进程2同样包含与PIPE的连接信息，然后每个进程关闭不需要的一个连接。

匿名管道是用打开的文件描述符来标识，命名管道是通过标识文件系统中的路径名来标识一个IPC通道，创建的FIFO文件在磁盘上没有数据块，仅用来标识内核中的一条通道.

管道的读写

写入函数在向管道中写入时，需要先检查VFS(Virtual Filesystem Switch)索引节点上的信息，同时满足下面的条件才可以进行内存复制操作：

  • 内存中有足够的空间容纳写入的数据
  • 内存没有被读程序锁定

消息队列

消息队列提供了一种从一个进程向另一个进程发送一个数据块的方法。

实现：msgget得到消息队列标识符或创建一个消息队列对象，msgsnd将消息写入消息队列，msgrcv从消息队列读取消息，msgctl获取和设置消息队列的属性

与命名管道相比：1. 消息队列也可以独立于发送和接收进程而存在，从而消除在同步命名管道的打开和关闭时产生的困难。2. 我们可以通过发送消息来避免有名管道的同步和阻塞问题。3. 可以通过消息类型选择性接受消息。

共享内存

把一片逻辑内存共享出来，让不同的进程去访问它，修改它，这是在两个运行的进程之间共享和传递数据的一种非常有效的方式。而这通常需要用其他的同步机制来同步对共享内存的访问，如信号量。

实现：通过shmget得到一个共享内存标识符或创建一个共享内存对象int shmget(key_t key, size_t size, int shmflg)，shmat将共享内存区对象映射到调用进程的地址空间void *shmat(int shmid, const void *shmaddr, int shmflg)，shmdt将断开共享内存连接，shmctl(共享内存管理)

自旋锁

自旋锁是一种Linux内核中广泛运用的底层同步机制。自旋锁是工作在多处理器环境的特殊的锁，如果是单处理环境中自旋锁会被替换为空操作。当某个处理器上的内核申请自旋锁时，如果锁可以用，则获得锁，然后执行临界区操作，最后释放锁；如果锁被占用，线程并不会转入睡眠状态，而是忙等待该锁，一旦锁被释放，则第一个感知此信息的线程获得该锁。

因此这有个特点就是线程一直占用cpu。相比较与基于OS休眠和唤醒机制的互斥锁更适用于临界区短、对CPU延时要求比较高的情况。
