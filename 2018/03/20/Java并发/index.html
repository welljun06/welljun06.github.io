Java并发

Posted on 2018-03-20

基本概念

进程：程序的执行过程，持有资源（共享内存，文件）和线程。实现是操作系统通过周期性地将CPU从一个进程切换到另外一个进程。

线程：线程是系统中最小的执行单元

线程交流方式：互斥和同步

并行：多个cpu或者多台机器同时执行一段处理逻辑

并发：通过调度算法，让用户看上去同时执行，从性能上的角度来看，如果没有任务会阻塞，那么在单处理器上使用并发是没有意义的。

线程安全：指的是在并发的情况下，该代码经过多线程使用，线程的调度不影响任何结果。这个时候使用多线程，我们只需关注系统的的内存、cpu是否够用。反之，线程不安全就意味着线程的调度顺序会影响最终的结果，

同步：人为的调度和控制，保证共享资源的多线程访问成为线程安全，来保证结果的正确性

线程的状态

Java线程一共有七个状态，分别是新建（new），就绪（Runnable），运行中（running），睡眠（sleeping），阻塞（Blocked），等待（waiting），死亡（Dead/Terminated）

新建：新创建一个线程对象

就绪：线程对象创建之后，其他线程调用了该方法的start()方法，该线程位于可运行线程池中，等待被线程调度选中分配cpu

运行中：可运行线程获得cpu后，执行代码片段

睡眠（TIMED_WAITING）：有等待时间的等待状态。

阻塞：指cpu因为某种原因放弃cpu使用权，暂时性地停止运行。直到线程进入可运行状态，才有机会获得cpu进入运行状态。

等待：执行wait方法后进入

死亡：线程的run()、main()方法执行完毕后，或者因为异常结束生命周期

run方法通常是会有某种形式的循环，使得任务一直循环下去直到不需要。

从任务中产生返回值

实现Callable接口，这是一个具有类型参数的泛型。它的类型参数表示的是从方法call()中返回的值。

休眠

执行sleep()使任务终止执行制定的时间

可以使用更加显式的版本TimeUnit.MILLISECONDS.sleep(time)，这允许指定时间单元，提供更好的可阅读性。

优先级

调度器倾向与让优先级高的线程先执行，但是优先级较低的只是执行的频率低，并不意味着不执行

可以在任务内部调用Thread.currentThread()来获得驱动该任务的Thread对象的引用

让步：通过调用yield()来建议运行其他具有相同优先级的线程

后台线程

是指程序运行的时候再后台提供一种通用服务的线程。当所有非后台程序结束的时候，程序也就会终结。

setDaemon()设置后台线程

后台创建的线程会自动被设置成后台线程

加入一个线程

一个线程可以在其他线程上调用join()方法，其效果是等待一段时间直到第二个线程结束才继续执行。

join方法也可以被中断，做法是在调用线程上调用interrupt()方法，这时需要调用try-catch子句。
