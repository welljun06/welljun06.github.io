Java并发

Posted on 2018-03-20 | In java

基本概念

进程：程序的执行过程，持有资源（共享内存，文件）和线程。实现是操作系统通过周期性地将CPU从一个进程切换到另外一个进程。

线程：线程是系统中最小的执行单元

线程交流方式：互斥和同步

并行：多个cpu或者多台机器同时执行一段处理逻辑

并发：通过调度算法，让用户看上去同时执行，从性能上的角度来看，如果没有任务会阻塞，那么在单处理器上使用并发是没有意义的。

线程安全：指的是在并发的情况下，该代码经过多线程使用，线程的调度不影响任何结果。这个时候使用多线程，我们只需关注系统的的内存、cpu是否够用。反之，线程不安全就意味着线程的调度顺序会影响最终的结果，

同步：人为的调度和控制，保证共享资源的多线程访问成为线程安全，来保证结果的正确性

线程的状态

Java线程一共有七个状态，分别是新建（new），就绪（Runnable），运行中（running），睡眠（sleeping），阻塞（Blocked），等待（waiting），死亡（Dead/Terminated）

新建：新创建一个线程对象只会短暂存在于这个状态，已经分配了必要的系统资源，并执行了初始化，已经有资格获得CPU时间了。

就绪：线程对象创建之后，其他线程调用了该方法的start()方法，该线程位于可运行线程池中，等待被线程调度选中分配cpu，在任意时刻，可以运行和不运行。

运行中：可运行线程获得cpu后，执行代码片段

睡眠（TIMED_WAITING）：有等待时间的等待状态。

阻塞：指cpu因为某种原因放弃cpu使用权，暂时性地停止运行。直到线程进入可运行状态，才有机会获得cpu进入运行状态。

等待：执行wait方法后进入

死亡：线程的run()、main()方法执行完毕后，或者因为异常结束生命周期

从任务中产生返回值

实现Callable接口，这是一个具有类型参数的泛型。它的类型参数表示的是从方法call()中返回的值。

Callabel 与 Runnable的区别

  1. callable 是具有返回值，具体的返回值是在callable的接口中的call返回的，并且这个返回值的具体值是通过实现Future接口的对象的get方法获取的，这个方法会造成线程阻塞；而Runnable接口中的run方法是没有返回值的。
  2. Callable里面的call方法可以抛出异常，我们可以使用捕获异常进行处理；但是Runnable中的run方法是不可以抛出异常的，异常需要在run方法内部进行处理，不能向外界抛出。

执行sleep()使任务终止执行制定的时间

优先级

调度器倾向与让优先级高的线程先执行，但是优先级较低的只是执行的频率低，并不意味着不执行

可以在任务内部调用 Thread.currentThread()来获得驱动该任务的Thread对象的引用

让步：通过调用yield()来建议运行其他具有相同优先级的线程

后台线程

是指程序运行的时候再后台提供一种通用服务的线程。当所有非后台程序结束的时候，程序也就会终结。

setDaemon()设置后台线程

后台创建的线程会自动被设置成后台线程

加入一个线程

一个线程可以在其他线程上调用join()方法，其效果是等待一段时间直到第二个线程结束才继续执行。

join方法也可以被中断，做法是在调用线程上调用 interrupt() 方法，这时需要调用 try-catch 子句。

中断

Thread类中包含interrupt()方法，可以终止被阻塞的任务。这个方法设置线层的中断状态。如果一个线程已经被阻塞，或者试图执行一个阻塞操作， 那么设置这个线程的中断状态将抛出InterruptedException异常。当抛出该异常或者是执行Thread.interrupted()时，中断状态将被复位。调用interrupted()相当于提供了离开run()循环而不抛出异常的方式。

检查中断

当你在线程上调用 interrupt() 时，中断发生的唯一时刻是在任务需要进入到阻塞操作中，或者已经在阻塞内部时。那么 run() 方法没有产生任何阻塞调用的时候，你的任务需要第二种方式来退出。这种机会是由中断状态来表示，其状态是通过调用 interrupt() 方法来设置，通过 interrupted() 来检查中断状态，这不仅可以告诉你 interrupt() 是否被调用，还可以清楚中断状态。而清楚中断状态可以确保并发结构不会就某个问题被中断提醒你两次。

在 blocked3上使用 try-finally 是为了使得无论 run() 是怎么退出，清理都会发生。

线程之间的的协作

现在问题不是彼此之间的干涉，而是彼此之间的协调。因为某些部分必须要在其他部分之前完成。

wait()与notifyAll()

wait() 使你可以等待某个条件发生变化，这会将任务挂起，只有在 notify 或者是 notifyAll 发生的时候，任务才会被唤醒并去检查所产生的变化，wait提供了一种任务之间对活动同步方式。

当一个方法调用wait的时候，线程的执行将会被挂起，对象上的锁被释放。就意味着其它的任务可以获得这个锁。

wait有两种形式

  • 一种是可以接受时间的，可以通过notify、notifyAll或者是时间到期来恢复执行
  • 另一种是不接受参数，无限wait下去，直到出现notify和notifyAll

这里使用while来包围wait()，因为：

  • 你可能有多个任务出于相同的原因在等待同一个锁，而第一个唤醒任务可能会改变这种情况，如果属于这种情况，那么这个任务应该被再次挂起。比如说使用了notifyAll()，但是2例子中的waxOn还没有被改变。
  • 在这个任务被唤醒的时候，有时候会有某个其他的任务已经做出了改变，从而使得这个任务在此时不能执行，或者执行结果已经无关紧要。此时应该通过wait再次让任务挂起。
  • 也要可能是某些任务出于不同的原因在等待你的对象上的锁，因此你需要检查是否已经由正确的原因进行唤醒，否则再次调用wait()。
