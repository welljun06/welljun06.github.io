Java注解

Posted on 2018-03-24

注解为我们在代码中添加信息提供一种形式化的方法，使我们可以在稍后某个时刻非常方便地使用这些数据。

内置Java.lang注解：

  • @Override，表示当前方法定义将覆盖超类中的方法，如果方法名对不上，编译器会报错
  • @Deprecated，如果程序员使用了注解为它的元素，那么编译器会发出警告信息
  • @SuppressWarning，关闭不当的编译器警告信息，

基本语法

从语法的角度来看，注解的使用方式几乎与修饰符的使用一模一样。

Java注解

与其他Java接口类似，注解也会编译成class文件。

  • @Target，定义注解将用于什么地方
  • @Rectetion，定义该注解在哪一个级别使用

Spring注解

  • @Autowired，根据类型的自动注入依赖（基于注解的依赖注入），可以被使用在属性域，方法，和构造函数上
  • @Qualifer，指定依赖的bean的id

Spring Boot注解

  • @Configuration ，表示是一个JavaConfig配置类
  • @EnableAutoConfiguration，将所有符合条件的bean定义加载到IOC容器
  • @ComponentScan，对应xml配置中的元素，自动扫描并加载符合条件的组件或者是bean的定义最终将这些bean定义加载到Ioc容器中。
